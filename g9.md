## FOLLOW Torture-Test Grammar

```
S  → A B C
A  → a A | ε
B  → b B | D | ε
C  → c C | ε
D  → d | ε
```

---

## Why this grammar is evil

### Nullable symbols

```
A, B, C, D are ALL nullable
```

So FOLLOW must propagate:

* from right to left
* through **multiple nullable symbols**
* repeatedly until a fixed point

---

## Where bugs usually happen

### 1️⃣ FOLLOW propagation chain

From:

```
S → A B C
```

* FOLLOW(C) gets FOLLOW(S)
* B is nullable → FOLLOW(B) gets FOLLOW(C)
* A is nullable → FOLLOW(A) gets FOLLOW(B)

This requires **multi-hop propagation**.

---

### 2️⃣ FIRST–FOLLOW interaction

For:

```
B → D
```

* FOLLOW(D) must include FOLLOW(B)
* Because D may be ε, FOLLOW(B) propagates *again*

---

### 3️⃣ Symbols at end of production

```
C → c C
```

* FOLLOW(C) includes FOLLOW(C) (self-loop)
* Must not infinite-loop
* Must still converge correctly

FOLLOW(S) = { $ }

## Explanation highlights

### FOLLOW(A)

* From `S → A B C`

  * FIRST(B C) = { b, d, c, ε }
  * Since B and C nullable → include FOLLOW(S)
* ⇒ `{ b, d, c, $ }`

### FOLLOW(B)

* From `S → A B C`

  * FIRST(C) = { c, ε }
  * C nullable → include FOLLOW(S)
* ⇒ `{ c, $ }`

### FOLLOW(D)

* From `B → D`

  * D at end → FOLLOW(B)
* ⇒ `{ c, $ }`
